---
title: Value Driven Development
---
Value driven development extends software engineering practices like TDD and BDD into the ethical domain. The content argues that software is never value-neutral and that development practices should make those values explicit, measurable, and communicable. Instead of only testing whether code works, the approach asks what the code is for and which values it serves, such as accessibility, privacy, or fairness. Public Badges is positioned as a mechanism for surfacing and validating those commitments. The topic is significant because it turns development workflow into a moral and social contract between creators and users. It also links technical practice to public accountability in a way that is reusable beyond the specific project.
